项目描述：

本项目包含两部分 一部分为从summo入稿获取数据 另一部分为分析数据并发现高分物件的潜在共性 

从业务流程上说 
1.抓取数据 
 1.1 网站为https://www.fn.forrent.jp/fn/main_r.action?id=1748324712985
 1.2 进入网页后点击会社间流通
 1.3 选择物件を探す
 1.4 选择エリアから
 1.5 选择东京
 1.6 在市郡区を選ぶ中 遍历每个市郡区 
 1.7 点击類似物件推定反響数 然后抓取推定反響数为10件以上/月的物件的数据
 1.8 账号密码写进.env文件
 1.9 抓取数据时可以通过脚本查看网页的结构

2.对数据进行结构化储存
 2.1采用合适的数据库 数据量预计在5000-10000左右
 2.2 数据库表头应该和summo入稿的表头保持对应 
 2.3 对于包含多项的的表头进行拆分 如：沿線/駅 住所 物件名・号室应拆分为沿線/駅 住所 物件名 号室四项
 2.4 数据库设计要考虑方便利用python进行分析的需求

3. 运用聚类分析， 分类树或其他方式查找高分物件的特性
 3.1 可以利用python的数据分析工具进行分析 找出高分物件共性
